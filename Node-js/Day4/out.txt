DATA ‚Üí JAB NAYA CHUNK AATA HAI

END ‚Üí JAB POORA DATA READ HO JATA HAI

ERROR ‚Üí JAB KOI ERROR HOTI HAI

FINISH ‚Üí JAB LIKHNA COMPLETE HOTA HAI

üîπ STEP-BY-STEP WORKING (READABLE STREAM EXAMPLE)
EXAMPLE CODE:
CONST FS = REQUIRE('FS');
CONST READSTREAM = FS.CREATEREADSTREAM('FILE.TXT');

READSTREAM.ON('DATA', (CHUNK) => {
  CONSOLE.LOG('NEW CHUNK:', CHUNK.TOSTRING());
});

READSTREAM.ON('END', () => {
  CONSOLE.LOG('FILE READING DONE!');
});

INTERNAL PROCESS üîç

FILE OPEN HOTI HAI:
FS.CREATEREADSTREAM() OS-LEVEL FILE HANDLE OPEN KARTA HAI.

BUFFER ALLOCATE HOTA HAI:
NODE EK INTERNAL BUFFER (DEFAULT ~64KB) BANATA HAI JISME FILE KA EK PART LOAD HOTA HAI.

CHUNK EMIT HOTA HAI:
JAB BUFFER FILL HOTA HAI, STREAM EK DATA EVENT EMIT KARTA HAI AUR TUMHE EK CHUNK DETA HAI.

FLOW CONTROL:
AGAR TUM DATA FAST PROCESS NAHI KAR PA RAHE HO, NODE STREAM KO PAUSE KAR DETA HAI AUTOMATICALLY.
JAB TUM READY HO JAATE HO, WO FIR SE RESUME KARTA HAI.

END EVENT:
JAB PURA FILE READ HO JAATA HAI ‚Üí END EVENT FIRE HOTA HAI.

CONST FS = REQUIRE('FS');
CONST WRITESTREAM = FS.CREATEWRITESTREAM('OUTPUT.TXT');

WRITESTREAM.WRITE('HELLO STREAM!');
WRITESTREAM.END();


WRITE REQUEST QUEUE HOTI HAI
NODE DATA KO EK INTERNAL QUEUE ME STORE KARTA HAI.

OS KO WRITE OPERATION BHEJA JAATA HAI (ASYNCHRONOUSLY).

JAB LIKHNA KHATAM HOTA HAI ‚Üí DRAIN AUR FINISH EVENTS TRIGGER HOTE HAIN.

üîπ PIPE WORKING INTERNALLY
READSTREAM.PIPE(WRITESTREAM);


PIPE() BASICALLY INTERNALLY DATA EVENT KO LISTEN KARTA HAI.

JAB BHI READSTREAM ME DATA EVENT AATA HAI ‚Üí WO US CHUNK KO WRITESTREAM.WRITE() ME BHEJ DETA HAI.

JAB READSTREAM KA END AATA HAI ‚Üí WRITESTREAM.END() CALL HOTA HAI AUTOMATICALLY.

SO:
PIPE() = CONNECT KARTA HAI SOURCE (READABLE) KO DESTINATION (WRITABLE) SE WITHOUT MANUAL EVENT HANDLING.

üîπ SUMMARY TABLE
STREAM TYPE	EXAMPLE	EMITS EVENTS	INTERNAL ROLE
READABLE	FS.CREATEREADSTREAM()	DATA, END	READ DATA IN CHUNKS
WRITABLE	FS.CREATEWRITESTREAM()	DRAIN, FINISH	WRITE DATA IN CHUNKS
DUPLEX	NET.SOCKET	BOTH	READ + WRITE
TRANSFORM	ZLIB.CREATEGZIP()	BOTH + TRANSFORM	MODIFY DATA ON THE FLY
üß† SIMPLE ANALOGY:

IMAGINE EK PIPELINE:

TAP (READABLE) ‚Üí PAANI CHHOTE FLOW ME DETA HAI (CHUNKS)

PIPE (TRANSFORM) ‚Üí PAANI FILTER KARTA HAI

BUCKET (WRITABLE) ‚Üí PAANI COLLECT KARTA HAI

NODE.JS Modified Streams BILKUL ISI TARAH ASYNCHRONOUSLY DATA PASS KARTE HAIN ‚Äî BINA SAARA DATA EK BAAR ME LOAD KIYE.

NODE.JS ME Modified Streams KA USE LARGE DATA KO CHHOTE-CHHOTE PARTS (CHUNKS) ME PROCESS KARNE KE LIYE HOTA HAI ‚Äî BINA POORA DATA MEMORY ME LOAD KIYE.
YE FAST AUR MEMORY-EFFICIENT HOTA HAI.

üîπ TYPES OF Modified Streams

READABLE STREAM ‚Üí DATA READ KARNE KE LIYE (EX: FS.CREATEREADSTREAM())

WRITABLE STREAM ‚Üí DATA LIKHNE KE LIYE (EX: FS.CREATEWRITESTREAM())

DUPLEX STREAM ‚Üí DONO (READ + WRITE) KAR SAKTA HAI (EX: NET.SOCKET)

TRANSFORM STREAM ‚Üí DATA KO MODIFY KARTE HUE PASS KARTA HAI (EX: ZLIB COMPRESSION)

üîπ EXAMPLE (FILE READ + WRITE STREAM)
CONST FS = REQUIRE('FS');

CONST READSTREAM = FS.CREATEREADSTREAM('INPUT.TXT');
CONST WRITESTREAM = FS.CREATEWRITESTREAM('OUTPUT.TXT');

// PIPE CONNECTS READ ‚Üí WRITE
READSTREAM.PIPE(WRITESTREAM);

CONSOLE.LOG('DATA COPIED SUCCESSFULLY!');
